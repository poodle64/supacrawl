# REST API Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Firecrawl v2-compatible REST API to Supacrawl with native extensions.

**Architecture:** FastAPI app served by uvicorn, launched via `supacrawl serve`. Reuses `SupacrawlServices` from MCP server for service lifecycle. Pydantic models handle camelCase/snake_case translation via aliases.

**Tech Stack:** FastAPI, uvicorn, Pydantic v2 (existing), httpx (existing, for tests)

**Spec:** `docs/superpowers/specs/2026-03-21-rest-api-design.md`

---

## Task 1: Foundation (app factory, auth, dependencies)

**Files:** Create `api/__init__.py`, `api/app.py`, `api/auth.py`, `api/dependencies.py`, `api/models/__init__.py`, `api/models/common.py`. Modify `pyproject.toml`.

- [ ] Add `api` optional extra to `pyproject.toml`: `fastapi>=0.115.0`, `uvicorn[standard]>=0.34.0`
- [ ] Create `api/app.py` with FastAPI app factory. Lifespan creates `SupacrawlServices` (reuse `create_supacrawl_services()` from `mcp/api_client.py`), stores on `app.state`, tears down on shutdown. Add CORS (all origins), add exception handler that maps all errors to `{"success": false, "error": "..."}`. Remap 422 to 400.
- [ ] Create `api/auth.py` with `get_api_key` FastAPI dependency. Reads `SUPACRAWL_API_KEY` from env. If set, validates `Authorization: Bearer <token>`. If unset, passes all requests.
- [ ] Create `api/dependencies.py` with helper to get services from `request.app.state`. Add factory for per-request `ExtractService` using shared `ScrapeService`.
- [ ] Create `api/models/common.py` with `ErrorResponse` model and base config (`populate_by_name=True`, `by_alias=True` for camelCase serialisation).
- [ ] Write test: `test_auth.py` - test auth with key set (valid/invalid), auth with key unset. Use FastAPI TestClient.
- [ ] Run tests, verify pass.
- [ ] Commit: `feat(api): add app factory, auth, and dependency injection`

## Task 3: Scrape endpoint

- [ ] Create Pydantic request model with camelCase aliases for all v2 scrape fields. Use `model_config = ConfigDict(populate_by_name=True)` and `Field(alias="onlyMainContent")` pattern. Accept and ignore unknown fields (`extra="ignore"`).
- [ ] Create Pydantic response model matching v2 shape (camelCase output). Map `raw_html` to `rawHtml`, `source_url` to `sourceURL`, `status_code` to `statusCode`, etc.
- [ ] Implement router: translate request to service params (including `maxAge` ms to seconds division, `proxy` mapping, `storeInCache` to `max_age`), call `ScrapeService.scrape()`, translate response.
- [ ] Write test: `test_scrape.py` - POST /scrape with camelCase body, verify camelCase response with correct shape.
- [ ] Write test: `test_translation.py` - verify unknown fields ignored, defaults applied, `maxAge` ms-to-seconds conversion.
- [ ] Run tests, verify pass.
- [ ] Commit: `feat(api): add POST /scrape endpoint`
