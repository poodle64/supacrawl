# web-scraper

Generic website ingestion pipeline that loads YAML-driven site configurations, runs crawls using Crawl4AI SDK (with optional Ollama integration), writes snapshot-based corpus layouts with manifests, and optionally chunks content to JSONL format for downstream LLM consumers. Supports fully local operation with Crawl4AI + Ollama for intelligent content extraction without external APIs.

## Features

- **YAML-driven configurations**: Declarative site configurations stored in `sites/` directory
- **Crawl4AI integration**: Uses Crawl4AI SDK for local web crawling
- **Snapshot-based output**: Each crawl creates a timestamped snapshot with manifest metadata
- **Optional chunking**: Convert snapshots to JSONL format for LLM consumption

## Overview

Web-scraper is a CLI tool that orchestrates website crawling using Crawl4AI. Site configurations define crawl parameters (entrypoints, include/exclude patterns, page limits), and web-scraper handles crawling execution, corpus snapshot creation, and optional content chunking.

### Architecture

```
Site Config (YAML) → Crawl4AI Scraper → Pages → Corpus Writer → Snapshot
                                                      ↓
                                                 Chunker → JSONL
```

Web-scraper **uses** the Crawl4AI SDK and does **not implement** scraping logic itself. All scraping is delegated to Crawl4AI.

### Directory Structure

- `web_scraper/` – core package (CLI, models, scrapers, corpus writer, chunker)
- `sites/` – user-provided site configuration YAML files
- `corpora/` – output snapshots (`<site_id>/<snapshot_id>/`)
- `docs/` – project documentation

## Quick Start

### Prerequisites

- Python 3.12+
- Conda (recommended) or Python virtual environment

### Development Environment Setup

1. **Create conda environment**:
   ```bash
   conda env create -f environment.yaml
   conda activate web-scraper
   ```

2. **Install dependencies**:
   ```bash
   pip install -e .
   ```

   **Note**: After installation, run `crawl4ai-setup` to install Playwright browser dependencies required by Crawl4AI. This is a one-time setup step.

3. **Set environment variables** (create `.env` from `.env.example` if needed):
   ```bash
   cp .env.example .env
   # Edit .env if you need to configure optional settings
   ```

4. **Verify installation**:
   ```bash
   web-scraper list-sites
   ```

## Usage

### Basic Commands

**List available site configurations**:
```bash
web-scraper list-sites
```

**Show site configuration details**:
```bash
web-scraper show-site meta
```

**Run a crawl** (creates `corpora/<site_id>/<snapshot_id>/`):
```bash
web-scraper crawl meta
```

**Chunk an existing snapshot**:
```bash
web-scraper chunk meta 2025-01-01_1200
```

**Discover URLs from sitemap** (without crawling):
```bash
web-scraper map meta
```

**Compress a snapshot** for archival:
```bash
web-scraper compress meta 2025-01-01_1200
```

### Common Workflows

**Initial setup**:
1. Create site configuration YAML file in `sites/` directory
2. Verify configuration: `web-scraper show-site <site-name>`
3. Run crawl: `web-scraper crawl <site-name>`
4. Check output in `corpora/<site_id>/<snapshot_id>/`

**Chunking for LLM consumption**:
1. Run crawl to create snapshot
2. Note snapshot ID from output (format: `YYYY-MM-DD_HHMM`)
3. Chunk snapshot: `web-scraper chunk <site-name> <snapshot-id>`
4. Use `chunks.jsonl` file in snapshot directory

**Chunking with Ollama summarization** (requires Ollama running locally):
1. Ensure Ollama is running on `localhost:11434`
2. Chunk with summaries: `web-scraper chunk <site-name> <snapshot-id> --use-ollama --ollama-summarize`
3. Chunks will include a `summary` field generated by Ollama

For an operator-focused overview of defaults, env toggles, and troubleshooting, see `docs/40-usage/USAGE_GUIDE.md`.

See [CLI Usage Guide](docs/40-usage/cli-usage-web-scraper.md) for complete command reference.

## Crawler

Web-scraper uses the Crawl4AI Python SDK (`crawl4ai`) running locally.

- **Installation**: The SDK is installed automatically via `pip install -e .`
- **Setup**: After installation, run `crawl4ai-setup` to install Playwright browser dependencies
- **Verification**: Run `crawl4ai-doctor` to verify installation
- **No API key required**: Runs entirely locally using Playwright browsers
- Uses `BFSDeepCrawlStrategy` for multi-page crawling
- **Magic Mode**: Enabled by default for better content extraction and anti-bot bypass
- **Ollama Integration**: Enable LLM-based content extraction by setting `CRAWL4AI_USE_OLLAMA=true`
  - Requires Ollama running on `localhost:11434`
  - Uses `OLLAMA_MODEL` environment variable (default: `llama3.2`)
  - Provides intelligent content extraction and cleaning
- Documentation: https://docs.crawl4ai.com/

### Quality-focused defaults

The scraper is pre-configured with sensible defaults that work well for most sites:

- **Stealth mode**: Random user agent, realistic viewport (1280x720), magic mode
- **Content extraction**: Pruning filter removes boilerplate, keeps main content
- **Rate limiting**: 2 requests/second with 1 second per-domain delay
- **Retry logic**: 3 attempts with exponential backoff

Most users don't need to change any settings. See `.env.example` for optional configuration.

### Ollama Integration

Optional LLM processing using local Ollama server for content enhancement. Used by both Crawl4AI (during crawling) and chunking (post-processing).

- **Requirements**: Ollama running on `localhost:11434` (default port)
- **Configuration**: Set `OLLAMA_HOST` and `OLLAMA_MODEL` environment variables (optional)
- **Crawl4AI Integration**: Enable LLM-based content extraction during crawling:
  - Set `CRAWL4AI_USE_OLLAMA=true` in environment
  - Crawl4AI will use Ollama for intelligent content extraction
  - Provides AI-powered content cleaning and extraction
- **Chunking Integration**: Enable with `--use-ollama` flag when chunking:
  - **Summarization**: Add summaries to chunks with `--ollama-summarize`
  - **Model selection**: Override default model with `--ollama-model`
- **Example (chunking)**: `web-scraper chunk <site-id> <snapshot-id> --use-ollama --ollama-summarize`
- **Example (crawling with Ollama)**: Set `CRAWL4AI_USE_OLLAMA=true`, then run `web-scraper crawl <site-name>`
- Documentation: https://ollama.com/

## Environment Variables

All configuration is optional - sensible defaults are built in. Copy `.env.example` to `.env` only if you need to change settings.

### Commonly Changed Settings

| Variable | Default | Description |
|----------|---------|-------------|
| `CRAWL4AI_HEADLESS` | `true` | Set to `false` to see the browser for debugging |
| `CRAWL4AI_ENTRYPOINT_TIMEOUT_MS` | `60000` | Timeout per page (increase for slow sites) |
| `CRAWL4AI_RATE_LIMIT_RPS` | `2.0` | Requests per second (lower = more polite) |
| `CRAWL4AI_RATE_LIMIT_DELAY` | `1.0` | Seconds between requests to same domain |

### Ollama Integration (Optional)

| Variable | Default | Description |
|----------|---------|-------------|
| `CRAWL4AI_USE_OLLAMA` | `false` | Enable Ollama for AI-powered features |
| `OLLAMA_HOST` | `http://localhost:11434` | Ollama server URL |
| `OLLAMA_MODEL` | `llama3.2` | Model to use for summaries |

See `.env.example` for advanced settings (browser pool, proxy rotation, content filtering thresholds).

## CLI Commands

| Command | Purpose |
|---------|---------|
| `web-scraper list-sites` | List available site configuration files |
| `web-scraper show-site <name>` | Show site configuration details |
| `web-scraper map <name>` | Discover URLs from sitemap without crawling |
| `web-scraper crawl <name>` | Run a crawl and create a snapshot |
| `web-scraper chunk <site> <snapshot>` | Chunk snapshot into JSONL for LLM consumption |
| `web-scraper compress <site> <snapshot>` | Compress a snapshot for archival |
| `web-scraper extract <archive>` | Extract a compressed snapshot archive |

### Crawl Options

```bash
web-scraper crawl <site-name> [OPTIONS]
  --verbose          Show crawl progress logs
  --resume <id>      Resume an existing crawl (use 'latest' for most recent)
  --rps <float>      Override requests-per-second rate limit
  --delay <float>    Override per-domain delay (seconds)
  --formats <list>   Output formats: markdown, html, text, json
```

### Chunk Options

```bash
web-scraper chunk <site-id> <snapshot-id> [OPTIONS]
  --max-chars <int>    Maximum characters per chunk (default: 1200)
  --use-ollama         Enable Ollama for AI processing
  --ollama-summarize   Add AI-generated summaries to chunks
  --ollama-model <m>   Override Ollama model
```
